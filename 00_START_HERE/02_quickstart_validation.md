# 02 — Quickstart Validation

## Objectif
Permettre à un lecteur externe de décider de manière binaire
si un artefact est transmissible.

---

## Test PASS / FAIL (lecture autonome)

### Condition 1 — Compréhension
Après lecture de l’artefact, le lecteur peut expliquer
avec ses propres mots :
- le but de l’artefact,
- à qui il s’adresse,
- ce qu’il permet de faire.

☐ PASS ☐ FAIL

---

### Condition 2 — Réutilisation
Le lecteur peut réutiliser l’artefact
dans un contexte similaire
sans poser de question à l’auteur.

☐ PASS ☐ FAIL

---

### Condition 3 — Vérifiabilité
Le lecteur peut vérifier seul
si l’artefact respecte les contraintes annoncées.

☐ PASS ☐ FAIL

---

## Décision finale

- ☐ PASS — l’artefact est transmissible
- ☐ FAIL — l’artefact n’est pas transmissible

---

## Action si FAIL
Retourner à la définition des contraintes
et itérer selon la logique opératoire.
